# NOAH v0.1 — 명세서 (Specification)

> **NOAH = Networked Operations Agent Hub**  
> 사람 + AI 에이전트를 위한 안정적 네이티브 메신저

**상태**: 명세 확정, 구현 시작 가능  
**작성**: 2026-04-13  
**버전**: v0.1

---

## 0. 핵심 가치 (Core Values)

```
1. 사용자 중심
   - 사용자가 자기 데이터의 진짜 주인
   - 저장/삭제/이동 모두 사용자가 결정

2. 안정성
   - 매일 사용 가능한 도구
   - 크래시 0회
   - 메시지 손실 0건
   
3. 단순함
   - 30초 가입
   - 직관적 UI
   - 마찰 최소화
   
4. AI 통합
   - @크루 멘션으로 AI 호출
   - 사람과 AI가 같은 채팅방
```

## 2. 데이터 모델

### 2.1 서버 DB (`noah_server.db`)

```sql
-- 사용자
CREATE TABLE users (
    user_id TEXT PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,        -- bcrypt
    display_name TEXT,
    avatar_url TEXT,
    status_message TEXT,
    created_at INTEGER NOT NULL,
    last_seen INTEGER
);
CREATE INDEX idx_users_username ON users(username);

-- 디바이스
CREATE TABLE devices (
    device_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    device_name TEXT,
    device_type TEXT,                   -- 'windows' / 'android'
    last_seen INTEGER,
    is_online INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- 친구 관계
CREATE TABLE friendships (
    user_id TEXT NOT NULL,
    friend_user_id TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    PRIMARY KEY (user_id, friend_user_id)
);

-- 메시지 큐 (오프라인 사용자용 임시)
CREATE TABLE pending_messages (
    msg_id TEXT NOT NULL,
    target_device_id TEXT NOT NULL,
    from_user_id TEXT NOT NULL,
    target_user_id TEXT NOT NULL,
    type TEXT NOT NULL,                 -- 'text' / 'file'
    payload TEXT NOT NULL,              -- JSON
    server_seq INTEGER NOT NULL,
    created_at INTEGER NOT NULL,
    expires_at INTEGER NOT NULL,        -- created_at + 7일
    PRIMARY KEY (msg_id, target_device_id)
);
CREATE INDEX idx_pending_target ON pending_messages(target_device_id);

-- 첨부 파일 임시 저장
CREATE TABLE pending_files (
    file_id TEXT PRIMARY KEY,
    msg_id TEXT NOT NULL,
    from_user_id TEXT NOT NULL,
    filename TEXT,
    mime TEXT,
    size INTEGER,
    storage_path TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    expires_at INTEGER NOT NULL
);

-- 글로벌 sequence
CREATE TABLE seq_counter (
    name TEXT PRIMARY KEY,
    value INTEGER NOT NULL DEFAULT 0
);
INSERT INTO seq_counter VALUES ('msg_seq', 0);
```

### 2.2 클라이언트 본체 DB (`system.db`)

```sql
-- 본인 정보
CREATE TABLE me (
    key TEXT PRIMARY KEY,
    value TEXT

    user_id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    display_name TEXT,
    avatar_url TEXT,
    status_message TEXT,
    last_active INTEGER,
    is_pinned INTEGER DEFAULT 0
);

-- 저장된 대화 인덱스 (.noahdb 파일들의 메타)
CREATE TABLE saved_conversations (
    conv_id TEXT PRIMARY KEY,
    title TEXT NOT NULL,                -- "크루와 대화" 또는 "김철수"
    file_path TEXT NOT NULL,            -- 사용자 지정 위치
    participant_user_ids TEXT,          -- JSON ["user_a", "user_b"]
    last_msg_text TEXT,
    last_msg_at INTEGER,
    msg_count INTEGER,
    file_size INTEGER,
    is_encrypted INTEGER DEFAULT 0,
    created_at INTEGER NOT NULL,
    last_opened_at INTEGER
);
CREATE INDEX idx_saved_last_opened ON saved_conversations(last_opened_at DESC);

-- 활성 대화 (앱 켜져 있는 동안만, 메모리 동기화용)
CREATE TABLE active_conversations (
    conv_id TEXT PRIMARY KEY,
    type TEXT NOT NULL,                 -- 'memory' (저장 안 함) / 'saved' (file 백업 중)
    file_path TEXT,
    is_modified INTEGER DEFAULT 0,
    opened_at INTEGER NOT NULL
);

-- 설정
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
-- 'auto_save_conversations' = 'ask' / 'always' / 'never'
-- 'default_save_folder' = '~/Documents/NOAH'
-- 'theme' = 'light' / 'dark' / 'auto'
-- 'notifications_enabled' = 'true' / 'false'
-- 'auto_start_with_os' = 'true' / 'false'
```

### 2.3 대화 파일 (`.noahdb`)

```sql
-- 메타데이터
CREATE TABLE conversation_meta (
    key TEXT PRIMARY KEY,
    value TEXT
);
-- 'conv_id', 'title', 'created_at', 'noah_version', 'participants'

-- 메시지
CREATE TABLE messages (
    msg_id TEXT PRIMARY KEY,
    server_seq INTEGER,
    from_user_id TEXT NOT NULL,
    from_username TEXT,
    text TEXT,
    has_attachment INTEGER DEFAULT 0,
    timestamp INTEGER NOT NULL,
    is_outgoing INTEGER NOT NULL,
    is_ai INTEGER DEFAULT 0,
    status TEXT NOT NULL,               -- 'pending', 'sent', 'received'
    created_at INTEGER NOT NULL
);

-- 첨부 파일 (BLOB 직접 저장 - 휴대성)
CREATE TABLE attachments (
    attachment_id TEXT PRIMARY KEY,
    msg_id TEXT NOT NULL,
    filename TEXT,
    mime TEXT,
    size INTEGER,
    data BLOB,
    created_at INTEGER NOT NULL
);

-- 검색 인덱스 (FTS5)
CREATE VIRTUAL TABLE messages_fts USING fts5(
    msg_id UNINDEXED,
    text,
    content='messages',
    content_rowid='rowid'
);
```

## 3. API 명세

### 3.1 인증

```
POST /api/auth/register
  Body: { username, password, display_name? }
  Response: { user_id, token }
  Error: 409 if username exists

POST /api/auth/login
  Body: { username, password }
  Response: { user_id, token, display_name }
  Error: 401 if invalid

POST /api/auth/logout
  Header: Authorization: Bearer <token>
  Response: 200 OK
```

### 3.2 사용자/프로필

```
GET /api/me
  Header: Bearer <token>
  Response: { user_id, username, display_name, avatar_url, status_message }

PUT /api/me
  Body: { display_name?, avatar_url?, status_message? }
  Response: 200 OK

POST /api/me/password
  Body: { old_password, new_password }
  Response: 200 OK
```

### 3.3 디바이스

```
POST /api/devices/register
  Body: { device_name, device_type }
  Response: { device_id }

GET /api/devices
  Response: [{ device_id, name, type, last_seen, is_online }]

DELETE /api/devices/{device_id}
  Response: 200 OK
```

### 3.4 친구

```
POST /api/friends/add
  Body: { username }
  Response: { user_id, username, display_name }

GET /api/friends
  Response: [{ user_id, username, display_name, last_active }]

DELETE /api/friends/{user_id}
  Response: 200 OK
```

### 3.5 메시지

```
POST /api/messages
  Body: {
    msg_id,                  // 클라가 생성한 ULID
    target_user_id,
    type,                    // 'text' / 'file'
    payload                  // 텍스트 또는 첨부 메타
  }
  Response: { server_seq, server_timestamp }

GET /api/messages/pending
  Header: Device-Id
  Response: [{ msg_id, from_user_id, type, payload, server_seq, ... }]

POST /api/messages/ack
  Body: { msg_ids: [...] }
  Response: { acked: N }
```

### 3.6 첨부 파일

```
POST /api/files/upload
  Multipart: file
  Response: { file_id, url, expires_at }

GET /api/files/{file_id}
  Response: 파일 다운로드
```

### 3.7 WebSocket

```
WS /ws

Client → Server:
  { type: "auth", token }
  { type: "ping" }

Server → Client:
  { type: "auth_ok" }
  { type: "new_message", msg_id, from, payload }
  { type: "pong" }
  { type: "friend_online", user_id }
  { type: "friend_offline", user_id }
```

### 3.8 AI 봇 (서버 자동 처리)

```
사용자 메시지에 "@크루"가 포함되면:
  1. 일반 메시지로 broadcast
  2. AI 봇이 해당 채팅방에 자동 참여
  3. Claude API 호출 (CPA Neowine)
  4. 응답을 메시지로 전송

AI 봇 ID: 'ai_crew' (고정)
표시 이름: '🕊️ 크루'
모델: claude-sonnet-4-6
```
