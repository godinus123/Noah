## 1. 컨텍스트 절약

### ❌ 피할 것
- 전체 파일 반복 읽기
- 같은 정보 여러 번 요청
- 긴 답변 장황하게
- 불필요한 장식/이모지 남발

### ✅ 권장
- 필요한 부분만 `view_range`로 읽기
- 파일 목록만 필요하면 `ls` 또는 `tree`
- 요약 먼저, 상세는 요청 시
- 응답은 핵심만

## 2. 파일 읽기 최적화

### 큰 파일
```bash
# ❌ 전체 읽기 (토큰 폭발)
cat Noah_design.md   # 1300줄

# ✅ 필요한 부분만
sed -n '1,50p' Noah_design.md
grep -n "섹션" Noah_design.md
head -100 Noah_design.md
```

### 여러 파일
```bash
# ❌ 하나씩 읽기
cat file1.md
cat file2.md
cat file3.md

# ✅ 한 번에 요약
wc -l *.md
ls -la | head
```
