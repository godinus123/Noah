## ❓ 카카오톡 API 연동 가능한가요?

### 짧은 답: **안 합니다.**

### 왜?

카카오 API는 **메시지 받기를 외부 앱에 허용하지 않습니다**.

| API | 가능 여부 | 비고 |
|-----|---------|------|
| 카카오 로그인 | ✅ | 인증만 |
| 카카오톡 "나에게 보내기" | ✅ | 본인에게만 |
| 카카오톡 친구에게 메시지 | ⚠️ | 친구가 앱 동의 필요, 단방향 |
| **카카오톡으로 메시지 받기** | ❌ | 제공 안 함 |
| 알림톡 (비즈니스) | ⚠️ | 사업자 등록, 유료, 템플릿 승인 |

### 대안: 공유 기능

NOAH는 카카오톡과 직접 연동하지 않지만, **OS 공유 기능**으로 우회합니다:

```
사용자: NOAH 메시지 → "공유" 버튼 → 카카오톡 → 친구 선택
사용자: 카카오톡 메시지 → 공유 → NOAH → AI에게 전달
```

자세한 내용: [`docs/sharing.md`](sharing.md)

### 향후 가능성

```
Phase 2 검토: 카카오 로그인 (가입 편의성)
  - 카카오 계정으로 NOAH 가입
  - 카카오 친구가 NOAH 사용자라면 자동 매칭
  - 단방향만 (카카오톡 ↔ NOAH 양방향 X)
```

## ❓ 드래그앤드롭, 폴더 첨부, Ctrl+C/V 가능?

### 짧은 답: **모두 가능, 무게 0 MB.**

### Windows (PC)

```
✅ 파일 드래그앤드롭 (NOAH 창에 끌어다 놓기)
✅ 폴더 드래그앤드롭 (재귀로 모든 파일)
✅ Ctrl+V 텍스트 (붙여넣기)
✅ Ctrl+V 이미지 (스크린샷, 캡처도구)
✅ Ctrl+V 파일 경로
✅ 파일 선택 다이얼로그
✅ 폴더 선택 다이얼로그
✅ 우클릭 → "NOAH로 보내기" (레지스트리 등록)
```

### Android (모바일)

```
✅ 파일 선택 (FilePicker)
✅ 갤러리 다중 선택
✅ 카메라 직접 촬영
✅ 다른 앱의 "공유" 메뉴에 NOAH 표시 (Share Intent)
✅ 클립보드 텍스트 붙여넣기
⚠️ 드래그앤드롭은 모바일에서 X (UX 자체가 다름)
```

### 라이브러리 추가 X

```
- WPF 드래그앤드롭: WPF 기본 (System.Windows.DragDrop)
- 클립보드: System.Windows.Clipboard
- 파일 다이얼로그: Microsoft.Win32.OpenFileDialog
- MAUI Share: Microsoft.Maui.ApplicationModel.Share
- MAUI FilePicker: Microsoft.Maui.Storage.FilePicker
- MAUI MediaPicker: 카메라/갤러리

모두 OS/프레임워크 기본 → +0 MB
```

자세한 코드: [`docs/sharing.md`](sharing.md)

## ❓ 사람도 사용할 수 있나요?

### 짧은 답: **물론입니다. NOAH는 사람 + AI 모두를 위한 메신저입니다.**

### 포지셔닝 정정

```
❌ 잘못된 이해: "AI 전용 메신저"
✅ 정확한 이해: "사람도 쓸 수 있는데, AI 통합이 일급 시민"
```

### 사람 사용자 vs AI 사용자

| 항목 | 사람 사용자 | AI 사용자 |
|------|-----------|---------|
| 인증 | 전화번호 + (옵션) SMS | API 토큰 |
| UI | 카카오톡 같은 GUI | 헤드리스 (CLI/SDK) |
| 메시지 입력 | 키보드/음성 | 프로그램 자동 |
| 응답 시간 | 분~시간 | 초~분 |
| 알림 | FCM/Toast | 상시 연결 |

### 차별점

```
일반 메신저:
  - 사람 ↔ 사람
  - 봇은 부착 (Slack 봇, Discord 봇)
  
NOAH:
  - 사람 ↔ 사람 (KakaoTalk 같은 UX)
  - 사람 ↔ AI (자연스러운 대화)
  - AI ↔ AI (협업)
  - 사람 + AI 그룹 채팅 (모두 함께)
```

NOAH의 가치는 **"AI 전용"이 아니라** "사람이 익숙한 메신저인데 AI도 같은 자격으로 참여"입니다.

## ❓ Phase 0부터 시작하는 이유는?

### 짧은 답: **빨리 동작 확인하기 위함.**

### 왜 작게 시작?

```
큰 프로젝트의 함정:
  - 3개월 설계 → 한 번에 작성 → 동작 안 함 → 멘붕
  - 너무 많은 종속성 → 디버깅 지옥
  - 동기 부여 떨어짐 (결과물이 안 보임)

NOAH 방식:
  - 3일 미니멀 → 동작 확인 → "와 됐다!"
  - 1주 기본 → 시연 가능
  - 1주 고급 → 카카오톡 같은 UX
  - 매 단계 결과물
```

### Phase 0 = 50줄짜리 서버 + 100줄짜리 클라

```
server.js: 50줄
MainWindow.xaml.cs: 80줄
MainPage.xaml.cs: 80줄

총 ~210줄로 "두 디바이스 사이 메시지 교환" 동작.
```

이게 되면 → 나머지는 점진적 추가.

자세한 계획: [`docs/phases.md`](phases.md)

## ❓ 공동 프로젝트인가요?

### 짧은 답: **네. 사람 + AI 모두 기여 가능.**

### 기여 방법

```
1. GitHub Issues에서 작업 선택
2. 자기 브랜치 생성
3. 작업 + 커밋 + 푸시
4. PR 생성 → 크루 리뷰
5. dev에 머지
```

### AI 에이전트도 동등 기여

```
크루:        설계, 조율, 코드 리뷰
안목:        모바일 빌드
비손피씨:    PC 빌드
비손서버:    서버 + 인프라
이효승:      비전, 의사결정, 테스트
```

각자 자기 영역에서 commit + push. PR 리뷰는 크루가.

## ❓ 이름 NOAH의 의미는?

### 이중 의미

#### 엔지니어링 (공식)
**NOAH** = **N**etworked **O**perations **A**gent **H**ub  
→ 분산 에이전트 통신을 위한 네트워크 운영 허브

#### 영감 (브랜딩)
창세기 8:11 — 노아가 비둘기를 보내 메시지를 받음.  
디지털 시대의 노아의 비둘기 = NOAH 메신저.

### 왜 두 가지?

- **개발자/엔지니어**: "Networked Operations Agent Hub"라는 기술적 의미
- **일반 사용자**: 비둘기 + 방주 친근한 브랜딩
- **마케팅**: 둘 다 활용 가능
